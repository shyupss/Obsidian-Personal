## Mental model

El **mental model** es la representación interna que tiene el usuario sobre cómo funciona un sistema. Es el “how users expect a system to work”.

*Lo que el usuario cree que está pasando.*

## Conceptual model

El **conceptual model** es el modelo diseñado por el sistema o producto para explicar cómo funciona la interacción. Es el “how designers structure the interaction”.

*Cómo el sistema está organizado para ser entendido.*

---
## Diferencia

#### Mental model
Está en la mente del usuario.
#### Conceptual model
Está en el diseño del sistema.

---
## Problema fundamental

Si el usuario piensa que:
- guardar = se guarda automáticamente

pero el sistema espera que:
- haga clic en “Save”

entonces hay desalineación.

El diseño debe intentar que el **conceptual model** del sistema sea compatible con el **mental model** del usuario.

*No se diseña para “cómo funciona técnicamente”, sino para cómo el usuario **cree** que debería funcionar.*
