La **Gestalt** es una teoría de la percepción que explica que el cerebro **no interpreta una interfaz como elementos aislados**, sino como **formas, grupos, patrones y estructuras completas**. En otras palabras, el usuario no “suma” visualmente botón + texto + caja + línea; más bien, percibe una organización global.

*Gestalt es una palabra alemana. Su traducción más aproximada es «totalidad», «patrón» o «forma». Implica que el significado no se encuentra al descomponer las cosas en partes, sino al apreciar el conjunto. En otras palabras, la Gestalt es un proceso holístico.*
## Por qué importa en UX

Cuando diseñas una interfaz, no controlas solo “qué elementos hay”, sino también **cómo el cerebro los agrupa**. Si agrupas mal, el usuario interpreta mal la estructura. Si agrupas bien, entiende rápido la interfaz.

Por eso Gestalt no es decoración: es una forma de **reducir esfuerzo cognitivo** y **hacer visible la estructura**.

## Principios clave

### Proximidad

Elementos cercanos se perciben como pertenecientes al mismo grupo.

Ejemplo:
- un label muy cerca de un input se entiende como asociado a ese input
- botones separados por mucho espacio parecen pertenecer a acciones distintas

Entonces, el espaciado comunica relaciones. Y el espacio en blanco no es necesariamente vacío: organiza.
### Similitud

Elementos parecidos en color, forma, tamaño o estilo se agrupan mentalmente.

Ejemplo:
- todos los botones azules parecen cumplir un rol similar
- tarjetas con el mismo formato se interpretan como parte de la misma categoría
### Continuidad

El ojo sigue líneas, trayectorias y patrones continuos.

Ejemplo:
- en una página, el usuario sigue una alineación vertical o una secuencia visual natural
- una línea o disposición alineada guía el recorrido
### Cierre

El cerebro completa figuras incompletas.

Ejemplo:
- logos incompletos que igual se entienden
- íconos simplificados que el usuario completa mentalmente

### Figura y fondo

El cerebro distingue un elemento principal y un fondo secundario.

Ejemplo:
- modal sobre fondo oscurecido
- CTA destacado respecto del resto

Entonces, lo importante debe sobresalir con claridad. Si fondo y contenido compiten, se pierde jerarquía.

### Simplicidad (Prägnanz)

El cerebro prefiere interpretaciones simples y estables.

Ejemplo:
- layouts limpios se entienden antes
- estructuras recargadas obligan a más esfuerzo

Simplificar no es “hacerlo bonito”, sino hacerlo más procesable.
### Focal point

Un elemento visual destaca primero.

Ejemplo:
- un botón principal
- un título destacado
- una imagen hero

Entonces, se debe decidir qué quieres que se vea primero. Si todo destaca, nada destaca.
### Common fate

Elementos que se mueven juntos se perciben como relacionados.

Ejemplo:
- tarjetas que se desplazan juntas
- elementos de una animación que aparecen coordinadamente

Por ende, el movimiento también agrupa.
