## Idea general

El diseño de experiencia de usuario (**UX Design**) es **interdisciplinario**, es decir, integra múltiples áreas de trabajo para diseñar experiencias efectivas, usables y satisfactorias.
## Disciplinas principales
### 1. User Research

Estudio de los usuarios para comprender sus necesidades, comportamientos y problemas.
- Métodos: entrevistas, encuestas, testing
- Objetivo: obtener evidencia para diseñar

> Sin research → diseño basado en suposiciones

---
### 2. Interaction Design

Diseño de cómo el usuario interactúa con el sistema.
- Flujos de usuario (user flows)
- Navegación
- Secuencia de acciones

> Define cómo se usa el sistema

---
### 3. Visual Design (UI)

Diseño visual de la interfaz.
- Colores
- Tipografía
- Layout
- Componentes

> Corresponde a la implementación visual de la interfaz

---
### 4. Information Architecture (IA)

Organización y estructuración eficaz del contenido .
- Jerarquías
- Menús
- Agrupación de información

> Define cómo se organiza el sistema

---
### 5. Usability Testing

Evaluación del sistema con usuarios reales.
- Identificación de errores
- Detección de problemas
- Mejora iterativa

> Diseño sin testing = suposición

---
## Relación entre disciplinas

Las disciplinas no funcionan de forma aislada:
- User research → identifica problemas
- Information architecture → organiza la solución
- Interaction design → define el flujo
- Visual design→ implementa la interfaz
- Usability testing → valida todo

---
## Rol del UX Designer

El UX Designer integra todas estas disciplinas y trabaja en conjunto con:
- Developers → implementación técnica
- Product Managers → objetivos de negocio
- Stakeholders → decisiones estratégicas

> UX está en la intersección entre usuario, negocio y tecnología

---
## Idea clave

UX no es una sola actividad, sino la integración de múltiples disciplinas que trabajan juntas para diseñar experiencias centradas en el usuario.
