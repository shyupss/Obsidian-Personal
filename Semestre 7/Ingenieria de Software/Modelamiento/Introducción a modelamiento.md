# ¿Qué es un modelo?

Un modelo es una **abstracción** que presenta una visión o perspectiva específica del sistema.
# Definición y alcance en modelado de sistemas

El modelamiento de sistemas es el proceso de desarrollo de un conjunto de **modelos abstractos**, donde cada uno ofrece una visión o perspectiva específica del sistema.

El estándar predominante es el ***Lenguaje de Modelado Unificado (UML)***, que utiliza notaciones gráficas.

> *También es posible el desarrollo de modelos formales matemáticos para especificaciones detalladas.*

---
# ¿Cuándo y para qué sirve el modelamiento?

Los modelos generados se usan:
1. **Durante el proceso de ingeniería de requerimientos** para ayudar a derivar los requerimientos de un sistema.
2. **Durante el proceso de diseño** para describir el sistema a los ingenieros que implementan el sistema.
3. **Después de la implementación** para *documentar la estructura y la operación del sistema*.

Estos se usan porque los sistemas reales son demasiado complejos.

---
# ¿Qué **perspectivas** del sistema se modelan?

Para una comprensión integral, se modelan cuatro perspectivas principales:
#### **Externa:**
Modela el contexto o entorno (Diagramas de Actividad).
#### **Interacción:**
Modela la relación entre el sistema y su entorno, o entre sus componentes (Diagramas de Caso de Uso).
#### **Estructural:**
Modela la organización y la estructura de datos (Diagramas de Clases, Componentes, Despliegue).
#### **Comportamiento:**
Modela la respuesta dinámica ante eventos (Diagramas de Secuencia).
