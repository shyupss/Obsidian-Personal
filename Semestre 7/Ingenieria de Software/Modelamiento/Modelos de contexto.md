Los modelos de contexto son el conjunto de representaciones que adoptan una ***perspectiva externa*** del sistema para definir sus alcances y su relación con el entorno.

> Estos se sitúan en un **nivel de análisis** (o punto de vista del negocio), lo que significa que su objetivo primordial es **entender el contexto y las necesidades** del usuario antes de decidir cómo se implementará técnicamente el software

Para modelar el contexto de un sistema, se pueden usar distintos diagramas. *Sin embargo estos no son excluyentes entre sí.*

> ***Cada diagrama aporta información distinta sobre el sistema.***

---
# Modelo de contexto simple

***Ayuda a los analistas a visualizar la separación entre el sistema bajo desarrollo y su ambiente.***

> Su objetivo principal no es mostrar cómo funciona el software por dentro, sino **determinar qué funcionalidades quedan dentro** del sistema **y cuáles pertenecen al entorno o dispositivos externos**
#### *Representación gráfica básica:
Se compone de un núcleo central que representa el sistema principal, rodeado de cajas que identifican los sistemas o dispositivos externos con los que este interactúa.*

# Diagrama de actividad UML

El **diagrama de actividad UML** se utiliza, dentro de la **perspectiva externa**, para representar el **Modelo de Proceso Empresarial**.

> El objetivo principal es mostrar el **flujo de trabajo** global, detallando cómo se mueve la información y las tareas dentro del proceso en el que el sistema está inserto.
