# ¿Qué es la ingeniería de software?
Es una disciplina - Aplicar teorías, métodos y herramientas donde es adecuado. Siempre trata de encontrar soluciones a problemas, incluso cuando no hay teorías ni métodos aplicables.

La ingeniería de software es una **disciplina** que busca **formalizar cómo se construye software** usando métodos, procesos y herramientas.

*“La Ingeniería del software es el estudio de los principios y metodologías para el desarrollo y mantenimiento de proyectos de desarrollo software (Zelkovitz, 1979).”*

No se trata de _hacer que funcione_, sino de **hacerlo bien, repetible y escalable**.
### Software como "capital del conocimiento"

Esto significa que el software no es sólo código. Son desiciones de diseño, conocimiento del dominio y experiencia acumulada.

# ¿Porqué existe la ingeniería de software?

En los años 60-70 se desarrollaba software sin tener en cuenta ningún tipo de parámetro de calidad:
- Software sin estándares
- Mala calidad
- Proyectos fallidos

Eso es lo que llamamos la “crisis del software”

Entonces, a causa de que los clientes buscaban cada vez más calidad nació la ingeniería de software como la conocemos hoy en día.

El **temita** es que el software es:
- intangible
- complejo
- difícil de medir

# Producto de software vs Sistema de software
#### Producto de software:
- Programa terminado
- Para muchos usuarios
- No personalizado
- Objetivo: **vender**
#### Sistema de software:
- Conjunto complejo de componentes
- Integración + arquitectura
- Hecho a medida o configurable
- Objetivo: resolver necesidades específicas

Con lo dicho, se puede decir que un sistema se compone de: Requisitos, arquitectura, integración y datos. Es decir, un producto de software es una pieza, pero un sistema de software es un ecosistema completo.

---
# Fundamentos para todo sistema de software

1. Debe llevarse a cabo usando un proceso de desarrollo **administrado** y **comprendido**. La organización que diseña el software necesita **planear el proceso de desarrollo.** 
2. La **confiabilidad** y el **desempeño** son importantes para todos los tipos de sistemas. No basta con que simplemente funcione.
3. Es importante **comprender y gestionar** la especificación y los **requerimientos** del software (lo que el software debe hacer). 
4. Tiene que usar de forma efectiva los recursos existentes. Se busca **eficiencia**.
