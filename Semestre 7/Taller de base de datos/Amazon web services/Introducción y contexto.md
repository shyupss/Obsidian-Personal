El panorama tecnológico ha experimentado un cambio radical desde la TI tradicional, donde una única base de datos relacional servía para todos los propósitos, hacia el uso de servicios especializados en la nube pública. El crecimiento exponencial de la generación de datos ha obligado a las organizaciones a evolucionar del manejo de Gigabytes (GB) a Petabytes (PB).

AWS ofrece múltiples servicios de bases de datos completamente gestionados, cada uno optimizado para casos de uso específicos, bajo la premisa de que no existe una solución única para todas las necesidades (_one-size-fits-all_).

**Resumen de Servicios de AWS por Tipo:**

|Tipo de Base de Datos|Servicio de AWS|Casos de Uso Comunes|
|---|---|---|
|**Relacional**|Aurora, RDS, Redshift|Aplicaciones tradicionales, ERP, CRM, comercio electrónico.|
|**Clave-Valor**|DynamoDB|Aplicaciones web de alto tráfico, sistemas de comercio, juegos.|
|**En Memoria**|ElastiCache (Memcached/Redis)|Almacenamiento en caché, gestión de sesiones, marcadores en tiempo real.|
|**Documentos**|DocumentDB|Gestión de contenido, catálogos, perfiles de usuario.|
|**Grafos**|Neptune|Detección de fraude, redes sociales, motores de recomendación.|
|**Libro Mayor (Ledger)**|QLDB|Sistemas de registro, cadena de suministro, transacciones bancarias.|
