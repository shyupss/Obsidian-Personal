Para seleccionar el servicio correcto, se deben responder preguntas críticas durante el proceso de arquitectura:

- **Naturaleza de los datos:** ¿Estructurados, semi-estructurados o no estructurados? ¿Esquema rígido o flexible?

- **Volumen y Duración:** ¿Qué cantidad de datos se almacenará y por cuánto tiempo?

- **Carga de Trabajo:** ¿Es una aplicación con predominancia de lectura, escritura o equilibrada?

- **Requisitos de Latencia:** ¿Se requieren respuestas en milisegundos?

- **Usuarios:** ¿Cuántos usuarios concurrentes se esperan?

- **Recuperación de Desastres:** ¿Cuáles son los objetivos RTO (tiempo de recuperación) y RPO (punto de recuperación)?
