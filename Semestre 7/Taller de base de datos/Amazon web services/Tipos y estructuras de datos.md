La elección de una base de datos depende fundamentalmente de la estructura del dato que se desea almacenar:

1. **Datos Estructurados:** Poseen un esquema fijo y definido. Se organizan típicamente en formato tabular (filas y columnas). Son ideales para consultas complejas y analítica. Ejemplo: Bases de datos relacionales gestionadas con SQL.

2. **Datos No Estructurados:** Carecen de un esquema definido y no encajan en hojas de cálculo tradicionales. Se caracterizan por no tener propiedades estructurales rígidas. Ejemplos: PDFs, archivos en sistemas como Amazon S3.

3. **Datos Semi-estructurados:** Un híbrido que no está constreñido por un esquema fijo pero posee organización interna (metadatos o etiquetas semánticas). Son ideales para aplicaciones de baja latencia y Big Data. Ejemplos: Formatos JSON, XML, CSV y bases de datos NoSQL.
