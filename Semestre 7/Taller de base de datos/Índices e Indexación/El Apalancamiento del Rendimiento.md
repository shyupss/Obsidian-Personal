El rendimiento en MySQL se logra mediante la interacción de tres factores: hardware, optimizaciones e índices. Sin embargo, su capacidad de impacto (apalancamiento) es desigual:
## Hardware
Es el nivel más bajo de apalancamiento. Aunque MySQL corre sobre hardware, simplemente comprar equipos más rápidos es una solución de "fuerza bruta".
## Optimizaciones
Incluyen algoritmos y estructuras de datos que permiten a MySQL usar el hardware eficientemente. Tienen un mayor apalancamiento que el hardware.
## Índices
Proporcionan el mayor y mejor apalancamiento. Son obligatorios para cualquier volumen de datos no trivial.

> *"Los índices son especiales porque el rendimiento no se puede lograr sin ellos... Sin índices, el rendimiento se limita a la fuerza bruta".*
